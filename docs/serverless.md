## 目录

# serverless

特点：

- 事件驱动
- 运行时计算

## 场景

对实时性要求不高的场景。

- 邮件和短信推送，app 信息推送
- webhook：如 github hook
- 数据统计分析
- 定时任务，爬虫，

如果需要提供不间断的，随时处理大量请求的服务，则不适合 serverless。

## 优缺点

- 优点
  - 便宜：云服务器需要一直运行， 采用函数计算的 Serverless 是按需计算。按需计算就意味着，在请求到来的时候，才运行函数。没有请求的时候，是不算钱的。
  - 快速部署
  - 快速开发
- 缺点
  - 冷启动
  - 完全依赖第三方服务
  - 缺乏调试和开发工具
